## Overview

**MetalQuery** is a Natural Language to SQL (NL2SQL) chatbot system that allows users to query an industrial furnace KPI database using plain English. The system also supports document-based Q&amp;A from BRD (Business Requirement Document) PDFs.

### Key Capabilities
| Feature | Description |
|---------|-------------|
| NL-to-SQL | Convert questions to PostgreSQL queries |
| BRD RAG | Answer questions from PDF documents |
| Hybrid Routing | Auto-detect SQL vs BRD questions |
| Security | Multi-layer SQL validation |

### Technology Stack
| Layer | Technology | Port |
|-------|------------|------|
| Frontend | React 18 | 5173 |
| Backend Gateway | Django 4.2 | 8000 |
| NLP Service | FastAPI + LangChain | 8004 |
| LLM Provider | Groq (llama-3.3-70b) | Cloud |
| Database | PostgreSQL | 5432 |
| Vector Store | ChromaDB (Multimodal) | Embedded |

### Multimodal RAG Capabilities
| Feature | Description |
|---------|-------------|
| Text Chunks | 961 chunks from 33 PDFs |
| Images | 389 extracted images with context |
| Image Filtering | Removes logos, icons, headers/footers |
| Lightbox Viewer | Click-to-expand image display |

## Component Details

### 1. React Frontend (`/frontend`)

**Purpose:** User interface for the chatbot

**Key Files:**
| File | Description |
|------|-------------|
| `src/App.jsx` | Main chat component |
| `src/config.js` | API URL configuration |
| `src/components/` | UI components |

**Features:**
- Chat message display with user/bot avatars
- Results table with formatted values
- SQL query viewer with copy button
- Loading states and error handling

---

### 2. Django Backend (`/backend`)

**Purpose:** Security gateway, database owner, request orchestration

**Key Files:**
| File | Description |
|------|-------------|
| `chatbot/views.py` | Chat endpoint, SQL validation, execution |
| `chatbot/urls.py` | API routes |
| `chatbot/relevant_models.py` | Table whitelist |
| `config/settings.py` | Django configuration |

**Responsibilities:**
| Role | Implementation |
|------|----------------|
| Rate Limiting | 30 requests/minute per IP |
| SQL Validation | Defense-in-depth (validates after NLP) |
| Query Execution | Safe execution with timeout |
| Audit Logging | All queries logged |
| Schema Provider | Exposes schema to NLP service |

---

### 3. NLP Service (`/nlp_service`)

**Purpose:** Natural language to SQL conversion (NO database access)

**Key Files:**
| File | Description |
|------|-------------|
| `main.py` | FastAPI app, hybrid routing |
| `config.py` | Environment configuration |
| `prompts_v2.py` | Schema-aware prompts, 29 tables |
| `guardrails.py` | SQL security validation |
| `query_router.py` | Routes SQL vs BRD queries |
| `brd_rag.py` | BRD document retrieval |
| `diagnostic.py` | SQL validation diagnostics |

**Endpoints:**
| Endpoint | Method | Purpose |
|----------|--------|---------|
| `/api/v1/chat` | POST | Hybrid (SQL + BRD) |
| `/api/v1/generate-sql` | POST | SQL generation only |
| `/api/v1/format-response` | POST | Format results |
| `/health` | GET | Health check |
